### 2. 从数学角度重新梳理了当前 MPPI 算法
今天系统梳理了当前代码对应的数学结构，包括：

- 状态与控制：
  - 状态为 `(x, y, theta)`
  - 控制为 `(v, omega)`

- 离散动力学：
  - 机器人按当前朝向前进
  - 同时允许角速度改变朝向

- rollout 的意义：
  - 一条控制序列经过动力学递推，会生成一条未来轨迹

- 当前优化问题的本质：
  - 在有限时域内，为未来 `horizon` 步寻找一条更优控制序列
  - 目标是在“接近目标、避障、安全、平滑”之间做平衡

- 当前 MPPI 的基本流程：
  1. 给定 nominal sequence
  2. 围绕 nominal sequence 采样很多候选控制序列
  3. rollout 成很多候选轨迹
  4. 用 cost 对每条轨迹打分
  5. 用指数权重对样本做加权平均
  6. 得到新的参考控制序列
  7. 只执行第一步，再进入下一轮 receding horizon

### 3. 理解了导师建议中“协方差矩阵”的作用
今天重点从线性代数和多元微积分角度理解了：

- 协方差矩阵不是单纯表示“噪声大不大”
- 它还能描述：
  - 噪声在不同方向上的扩散强弱
  - 多维噪声云团的方向和形状
- 在二维情况下：
  - 若协方差矩阵各方向相同，则噪声云团更像圆
  - 若不同方向扩散不同，则云团更像椭圆
- 因此，协方差矩阵可以用来表达“沿某些方向更愿意探索、沿另一些方向更保守”

### 4. 理解了为什么导师建议会让噪声和环境相关
今天明确了一个关键点：

不是因为“用了协方差矩阵”，噪声就自动和环境相关；  
而是因为我们把协方差矩阵设计成了当前位置和局部环境几何的函数。

也就是说，如果写成：

- `Sigma(x)`
- 或 `Sigma_t(x_t)`

那么不同位置会对应不同的噪声形状、方向和强度。

进一步理解了导师建议里的思路：

- 用 signed distance field 的梯度给出法向方向
- 用与法向正交的方向作为切向
- 再构造：
  - 沿切向更大的扩散
  - 沿法向更小的扩散

这样就能让采样更倾向于“顺着 corridor 走”，而不是“朝障碍方向乱撒”。

### 5. 明确了当前代码与导师建议之间的差距
今天还确认了当前代码里的采样函数本质上仍然是：

- 对 `v` 单独加高斯扰动
- 对 `omega` 单独加高斯扰动

也就是说，当前采样仍然是固定参数、与环境无关的高斯采样。

因此，导师提出的“位置相关协方差噪声”目前还没有真正接入到现有实现里。
