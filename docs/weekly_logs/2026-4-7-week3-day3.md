# 2026-04-07 MPPI / MuJoCo 进展

## 今日主线
继续沿着“二维/平面导航 planner 保持不动，MuJoCo 负责执行平台与显示层”推进，没有转向真正三维规划问题。

## 已确认的阶段状态
- anisotropic sampling 主线继续保留，`sdf_influence_distance=1.2` 作为当前默认值。
- MuJoCo viewer 与实验入口保持可运行。
- dense 场景下，今天主要工作集中在 MuJoCo 侧的场景显示、边界相关行为观察、以及预测轨迹在线可视化。

## 今天完成的内容
1. 继续完善 MuJoCo 场景显示层：
   - 调整了场景内轨迹可视化相关的 XML 预埋显示点；
   - 调整了轨迹颜色、点数和显示层级，让路径在 viewer 中更容易观察。
2. 在 MuJoCo 环境层接入 predicted trajectory 在线显示：
   - 能在 viewer 中实时看到 best predicted trajectory；
   - 后续也扩展到了部分辅助预测路径显示，用于观察 sample cloud 的走向。
3. 借助在线可视化定位了当前失败机理：
   - 机器人在关键障碍附近时，sample / predicted path 会明显朝边界方向偏；
   - safety filter 会把越界动作拦住，因此机器人先在边界附近磨蹭；
   - 部分 case 中，机器人会先停顿较长时间，之后再重新找到路；
   - 也有 case 会在特定位置停住直到 `execute_steps` 结束。
4. 因此，当前问题判断已经从“单纯边界罚不够”转为：
   - proposal distribution / sampling bias 在局部位置发生偏移；
   - 同时存在“安全但不前进”的 stall / freeze 局部最优。

## 今日结论
- 单纯继续增大边界推力，效果有限；
- 当前更值得优先处理的是：
  1. sampler 在障碍/边界附近的方向偏置；
  2. cost 中缺少对“低位移、低进展、空转/停滞”的显式惩罚。

## 明日建议起点
1. 先在 `trajectory_cost(...)` 中加入轻量 anti-stall / progress-delta 项；
2. 只跑最典型的 freeze case，观察：
   - 第一个拐弯处磨蹭步数是否下降；
   - 中后段是否还会停到 `execute_steps` 结束；
3. 如果 stall 现象仍明显，再继续改 weighted update / elite update，而不是盲目继续加大 boundary push。

## 备注
- 今日重点是“看清 failure mode”，不是做大规模 sweep。
- 当前 MuJoCo 可视化已经足够支撑后续对 sample cloud 和局部停滞机理的定位分析。
